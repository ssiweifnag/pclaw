## 2. 技術原理

### 2.1 WiFi CSI（Channel State Information）
- WiFi 信號在傳播過程中會被人體反射和遮擋
- CSI 記錄了信號的**振幅**和**相位**變化
- 通過分析這些變化，可以推斷人體的位置和姿態

### 2.2 核心技術流程
```
WiFi 發射器 → 信號傳播 → 人體反射 → WiFi 接收器
                                    ↓
                           CSI 數據採集
                                    ↓
                           深度學習模型（如 CNN/RNN）
                                    ↓
                           人體姿態/位置輸出
```

### 2.3 關鍵技術組件

| 組件 | 說明 |
|------|------|
| **WiFi 設備** | 需支援 CSI 提取的網卡（如 Intel 5300、NVIDIA Links） |
| **CSI 提取工具** | Linux 802.11n CSI Tool, Nexmon CSI |
| **信號處理** | 濾波、降噪、特徵提取 |
| **深度學習模型** | CNN（卷積神經網絡）、RNN（循環神經網絡） |
| **姿態重建** | 從 2D CSI 特徵重建 3D 人體姿態 |

## 3. 主要功能

### 3.1 典型應用
1. **人體存在檢測**（Through-wall Presence Detection）
2. **姿態估計**（Pose Estimation）
   - 站立、坐姿、行走
   - 骨架關鍵點定位
3. **追蹤**（Tracking）
   - 室內人員位置追蹤
   - 穿牆追蹤（需較近距離）
4. **生命體徵監測**
   - 呼吸頻率檢測
   - 心率估算（可選）

### 3.2 與傳統攝像頭的對比

| 特性 | WiFi CSI | 傳統攝像頭 |
|------|----------|-----------|
| 隱私侵犯 | 無（無圖像採集） | 高（直接拍攝） |
| 光照要求 | 無（WiFi 信號） | 需要光線 |
| 遮擋穿透 | 可穿牆/障礙物 | 無法穿透 |
| 計算成本 | 中等（需 ML 模型） | 低（直接識別） |
| 準確度 | 較低（對多人場景挑戰大） | 高 |

## 4. 對智慧建築的應用價值

### 4.1 建築領域相關性
- **BEMS（建築能源管理系統）**：根據人員存在調整 HVAC
- **IAQ（室內空氣品質）**：結合 CO₂ 感測，實現需求控制通風（DCV）
- **安全監控**：非侵入式人員監測

### 4.2 結合 Pclaw 研究方向
- 博士論文題目：**亞熱帶氣候區建築IAQ提升與節能整合效益研究**
- WiFi CSI 可以作為**室內人員存在感測**的低成本方案
- 可與現有 ESP32/LoRa 感測網絡整合

## 5. 技術限制與挑戰

1. **多人場景複雜度**：當有多人同時在場時，CSI 信號疊加難以分離
2. **設備要求**：需特定 WiFi 網卡，消費級路由器難以支援
3. **環境敏感性**：牆壁材質、家具擺設會影響信號傳播
4. **實時性**：深度學習推斷需要一定計算資源

---

## 6. 結論

WiFi 穿牆感知技術（RuView 類型）是 IoT 和智慧建築領域的前沿技術：
- **非侵入式**：不侵犯隱私
- **穿牆能力**：適用於障礙物場景
- **低成本**：利用現有 WiFi 基礎設施

對於智慧建築研究，結合 WiFi CSI 與現有 BEMS/IAQ 系統是一個值得探索的方向。
