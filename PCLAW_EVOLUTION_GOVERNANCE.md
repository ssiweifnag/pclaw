## 版本落後策略（落後2代原則）

### 版本對應關係

| 我的系統 | Pclaw 部署版 | 說明 |
|----------|--------------|------|
| v3.0 | - | 最新功能 |
| v2.0 | - | 開發完成 |
| v1.0 | v1.0 | 同步落後2代 |
| v0.9 | - | 測試中 |
| v0.8 | v0.8 | 已發布 |

### 為什麼落後2代？

1. **穩定性**：讓社群測試，發現問題
2. **可控性**：用戶有時間審批每個版本
3. **信任**：確保發布內容經過驗證
4. **安全**：避免敏感功能外流

## 反饋機制（開源貢獻）

### 問題/功能反饋流程

```
社群用戶
    ↓ 提交 Issue/PR
Pclaw 部署版（接收）
    ↓ 自動分類
用戶系統（審批）
    ↓ 審批通過
我的系統（整合）
    ↓ 落後2代發布
Pclaw 部署版（更新）
```

### 反饋類型

| 類型 | 標題格式 | 處理方式 |
|------|----------|----------|
| Bug | `[Bug] 問題描述` | 緊急 → 快速審批 |
| 功能請求 | `[Feature] 功能描述` | 評估 → 决定是否開發 |
| 技能分享 | `[Skill] 技能名稱` | 測試 → 審批發布 |
| 文檔改進 | `[Docs] 改進內容` | 快速審批 |

### 自動反饋配置

```yaml
# Pclaw 反饋設定
feedback:
  auto_report: true
  report_channel: "用戶系統"
  report_format: "issue_template"
  
  auto_labels:
    bug: "needs-review"
    feature: "under-evaluation"
    skill: "skill-submission"
    
  auto_assign:
    - "Panboo"
```

## 版本審批流程

### 發布前檢查清單

```markdown
## 版本發布審批清單

### 必須檢查
- [ ] 所有新技能已測試
- [ ] 沒有安全漏洞
- [ ] 向後兼容性
- [ ] 文檔完整

### 用戶審批
- [ ] 用戶確認版本內容
- [ ] 用戶批准發布
- [ ] 用戶設定發布時間

### 發布後
- [ ] GitHub Release 建立
- [ ] 版本標籤設置
- [ ] 更新 CHANGELOG.md
```

### 版本命名規則

```bash
# 版本格式：v{major}.{minor}.{patch}
# - major: 架構變更（需完全審批）
# - minor: 新功能（需審批）
# - patch: Bug 修復（自動 + 快速審批）

# Examples:
v1.0.0  # 正式版
v1.1.0  # 新功能
v1.1.1  # Bug 修復
```

## 技能開發與審批

### 技能分級

| 等級 | 來源 | 審批需求 | 發布位置 |
|------|------|----------|----------|

```markdown

### 必須滿足
1. 功能穩定（無明顯 Bug）

3. 實用性（對用戶有價值）
4. 文檔完整（有使用說明）

### 加分項
1. 開源許可（MIT/Apache）
2. 測試覆蓋
3. 效能優化
4. 多語言支持
```

## 疑難排解

| 問題 | 解法 |
|------|------|
| 社群提交惡意程式 | 沙箱測試 + 用戶審批 |
| 版本落後太多 | 緊急發布通道（用戶審批）|
| 技能衝突 | 版本隔離 + 衝突檢測 |
| 用戶隱私洩露 | 本地處理 + 最小化上傳 |
