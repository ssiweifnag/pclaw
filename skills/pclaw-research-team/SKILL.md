# 🎓 Pclaw 研究團隊技能

**版本：** v1.0
**日期：** 2026-03-30
**基於：** ai-design-team 多角色概念

---

## 簡介

Pclaw 研究團隊技能 — 借鑒 AI 設計部門的多角色分工模式，為博士論文研究打造的虛擬團隊。

---

## 👥 研究團隊角色

| 角色 | 核心職責 | 何時呼叫 |
|------|----------|----------|
| 🎓 研究總監 | 任務分派、方向把關、進度追蹤 | 不知道從哪開始 |
| 📚 文獻調研員 | 搜尋、篩選、整理文獻 | 需要文獻回顧 |
| 📊 數據分析師 | 數據處理、統計分析、圖表製作 | 有數據要分析 |
| ✍️ 論文寫作員 | 各章節撰寫、潤飾、格式調整 | 要寫論文 |
| 🔍 批判審核員 | 找漏洞、邏輯審查、改進建議 | 論文初稿完成 |
| 📋 APA 核查員 | 引用格式、參考文獻檢查 | 交稿前 |

---

## 🔄 研究工作流程

### 完整論文研究流程

```
① 研究總監     → 定義研究問題、拆解任務
② 文獻調研員   → 搜尋整理文獻（Zotero/arXiv/Google Scholar）
③ 數據分析師   → SenseLab 數據處理、統計分析
④ 論文寫作員   → 各章節撰寫（緒論→方法→結果→討論→結論）
⑤ 批判審核員   → 邏輯審查、漏洞修補
⑥ APA 核查員   → 引用格式核查、參考文獻完整性
```

### 單一任務流程

```
① 研究總監 → 確認任務性質
② 執行者   → 完成任務
③ 批判審核員 → 審查結果
```

