## 簡介

---

## 🏗️ DeerFlow 協調層（新增 v1.1）

借鑒 **DeerFlow**（⭐52k，ByteDance 開源）的 Multi-Agent 協調機制：

### 協調器職責

| 功能 | 說明 |
|------|------|
| **任務分類** | 識別任務類型（研究/程式碼/創作/分析） |
| **子任務分解** | 將複雜任務拆解為可並行的子任務 |
| **代理調度** | 根據任務類型指派最合適的專業代理 |
| **結果整合** | 彙整各代理輸出，去重、排序、生成最終答覆 |

### 協調流程

```
用戶輸入
    ↓ 任務分類
① 研究任務 → 文獻調研員 + 研究總監
② 分析任務 → 數據分析師 + 研究總監
③ 寫作任務 → 論文寫作員 + 批判審核員
④ 核查任務 → APA 核查員
    ↓ 結果整合（協調器）
最終輸出
```

### 與 DeerFlow 的架構對照

| DeerFlow 元件 | Pclaw 對應 |
|--------------|-----------|
| Coordinator | 研究總監（兼協調器） |
| Sub-agents | 各專業角色 |
| Memory | 記憶系統（M蒙ORy.md + 每日日誌） |
| Sandbox | （預留：安全代碼執行環境） |
| Tools | OpenClaw 工具集 + 技能系統 |
| Message Gateway | 協調器角色 |

## 👥 研究團隊角色

| 角色 | 核心職責 | 何時呼叫 |
|------|----------|----------|
| 🎓 研究總監 | 任務分派、方向把關、進度追蹤、協調各代理 | 不知道從哪開始 |
| 📚 文獻調研員 | 搜尋、篩選、整理文獻 | 需要文獻回顧 |
| 📊 數據分析師 | 數據處理、統計分析、圖表製作 | 有數據要分析 |
| ✍️ 論文寫作員 | 各章節撰寫、潤飾、格式調整 | 要寫論文 |
| 🔍 批判審核員 | 找漏洞、邏輯審查、改進建議 | 論文初稿完成 |
| 📋 APA 核查員 | 引用格式、參考文獻檢查 | 交稿前 |

## 🔄 研究工作流程

### 完整論文研究流程

```
① 研究總監     → 定義研究問題、拆解任務
② 文獻調研員   → 搜尋整理文獻（Zotero/arXiv/Google Scholar）
③ 數據分析師   → SenseLab 數據處理、統計分析
④ 論文寫作員   → 各章節撰寫（緒論→方法→結果→討論→結論）
⑤ 批判審核員   → 邏輯審查、漏洞修補
⑥ APA 核查員   → 引用格式核查、參考文獻完整性
```

### 單一任務流程

```
① 研究總監 → 確認任務性質（分類 + 分解）
② 執行者   → 完成子任務
③ 批判審核員 → 審查結果
④ 協調器   → 整合輸出
```
