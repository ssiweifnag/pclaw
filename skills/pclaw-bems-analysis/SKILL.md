## 1️⃣ sensor-data-fetch（建築感測器數據抓取）

### 輸入（Input）
- **CO₂：** MH-Z19B 感測器（ppm）
- **溫濕度：** SHT31 / DHT22（°C / %RH）
- **PM2.5：** PMSA003I（μg/m³）
- **電力：** INA228 / 智慧電表（W / kWh）
- **VOC：** SGP30（ppb）

### 處理（Process）
```
原始數據 → 清洗 → 對齊 → 補缺 → 去異常 → 標準化
```

### 輸出（Output）
- **標準化時序數據表**
- **數據品質報告**
- **異常標記**

### KPI
- 即時性（< 1 分鐘延遲）
- 穩定性（> 99% 在線率）
- 完整性（> 95% 數據覆蓋）

---

## 2️⃣ energy-baseline-parser（能耗基線解析）

### 輸入（Input）
- **建築基本資料：** 面積、用途、 occupancy
- **歷史能耗：** 月度 / 日度 / 小時級
- **天氣數據：** 溫度、濕度、焓值

### 處理（Process）
```
能耗數據 → 同期對比 → 歸一化處理 → 基線建立 → 偏差分析
```

### 輸出（Output）
- **能耗基線模型**
- **預測區間（±σ）**
- **節能潛力評估**

### 對應博士論文
- 既有建築能源檢測
- ASHRAE 90.1 基準對比

---

## 3️⃣ iaq-alert-scan（IAQ 異常事件掃描）

### 輸入（Input）
- **即時感測數據：** CO₂、PM2.5、VOC
- **門禁數據：** 人員進出（iClass）
- **行事曆：** 課程 / 會議排程

### 處理（Process）
```
即時數據 → 去噪 → 閾值偵測 → 趨勢分析 → 事件分類
```

### 輸出（Output）
- **情緒溫度計：** IAQ 等級（良好 / 普通 / 警告 / 危險）
- **事件清單：** 異常類型、時間、持續時間
- **預警提醒：** LINE / Telegram 通知

### 閾值設定
| 參數 | 良好 | 普通 | 警告 | 危險 |
|------|------|------|------|------|
| CO₂ (ppm) | < 600 | 600-800 | 800-1000 | > 1000 |
| PM2.5 (μg/m³) | < 12 | 12-35 | 35-55 | > 55 |
| Temp (°C) | 22-25 | 20-26 | 18-28 | <18 或 >30 |

---

## 4️⃣ pii-score-engine（PII 指標評分引擎）

### 因子池（Factor Pool）
| 因子 | 說明 | 權重 |
|------|------|------|
| IAQ-PMV | 室內空氣品質指標 | 30% |
| 熱舒適 | ASHRAE 55 熱舒適指標 | 20% |
| 能耗效率 | 每人每小時能耗 | 25% |
| 人員密度 | 每人大樓面積 | 15% |
| 通風效率 | CO₂ 換氣次數 | 10% |

### 處理（Process）
```
標準化 → 加權計算 → 行業基準對齊 → 分數輸出
```

### 輸出（Output）
- **PII 總分（0-100）**
- **分層標籤：** 優異 / 良好 / 一般 / 需改善
- **雷達圖：** 多維度視覺化

### 對應博士論文
- PII 指標（Performance Improvement Index）
- 32% IAQ 改善

---

## 5️⃣ energy-event-impact（能耗事件影響分析）

### 事件類型
| 類型 | 說明 | 影響期限 |
|------|------|----------|
| 課程結束 | 人員突然離開 | 短期（< 30 min）|
| 假期模式 | 建築低負荷 | 中期（1-7 days）|
| 天氣突變 | 冷氣 / 暖氣負載變化 | 短期（< 2 h）|
| 需求反應 | VRB / GEB 事件 | 中期（2-4 h）|

### 處理（Process）
```
事件識別 → 情景拆解 → 傳導鏈路分析 → 影響評估
```

### 輸出（Output）
- **短 / 中 / 長期影響判斷**
- **節能 / 耗能預測**
- **建議動作**

---

## 6️⃣ iaq-guardrail（IAQ 安全閾值護欄）

### 指標體系
| 指標 | 計算方式 | 閾值 |
|------|----------|------|
| CO₂ 峰值 | 每小時最大 ppm | < 1000 ppm |
| PM2.5 日均 | 24 小時平均 | < 35 μg/m³ |
| 溫度超限 | 每小時超限時間 | < 5% |
| 相對濕度 | 舒適區間 | 40-60% RH |

### 處理（Process）
```
閾值預警 → 報警觸發 → 通知發送 → 事件記錄
```

### 輸出（Output）
- **風險儀表板：** 紅 / 黃 / 綠燈
- **防禦建議：** 通風加強、人員疏散
- **事件報告：** 完整事件時間線

---

## 7️⃣ control-suggestion（控制策略建議）

### 輸入
- **PII 評分**
- **能耗基線**
- **人員預測**
- **天氣預報**

### 處理
```
數據融合 → 優化求解 → 策略生成 → 約束校驗
```

### 輸出（Output）
| 建議類型 | 說明 |
|----------|------|
| DCV 建議 | CO₂ 基礎通風控制 |
| 照明建議 | 人員感測照明控制 |
| 空調建議 | 焓值控制策略 |
| 峰值削減 | 需求反應建議 |

### 對應博士論文
- 需求控制通風（DCV）
- 焓值控制
- 17.2% 節能
